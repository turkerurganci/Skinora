## Yapılan İşler

- **`IntegrationTestBase` refactor** — `INTEGRATION_TEST_SQL_SERVER` env var varsa (CI: job-level shared SQL Server) o connection string server-level base olarak kullanılır; yoksa TestContainers ile tek bir ephemeral `MsSqlContainer` assembly başına lazy singleton olarak açılır (lokal/dev fallback). Her test class `InitializeAsync`'da `T_{ClassName}_{Guid:N}` formatında unique bir DB yaratır, `EnsureCreatedAsync` + seed çalıştırır; `DisposeAsync` `SqlConnection.ClearAllPools()` + `ALTER DATABASE ... SET SINGLE_USER WITH ROLLBACK IMMEDIATE; DROP DATABASE` ile temizler.
- **CI shared SQL Server** — `.github/workflows/ci.yml` integration-test job'una `services: mssql: mcr.microsoft.com/mssql/server:2022-latest` eklendi (health-check sqlcmd ile, start period 30 sn, 20 retry). Job-level `env.INTEGRATION_TEST_SQL_SERVER` ile connection string tüm test assembly'lerine geçer.
- **Paralel execution açıldı** — `backend/tests/xunit.runner.json`: `parallelizeTestCollections: true`, `parallelizeAssembly: true`. CI integration step'inden `-m:1` MSBuild flag'i kaldırıldı. Default paralel execution geri.
- **Thread-safe module registration** — `AppDbContext.RegisterModuleAssembly` artık `_moduleAssembliesLock` ile korunuyor; `OnModelCreating` `_moduleAssemblies` snapshot'ını lock altında alıyor. Paralel test runner'ın ortaya çıkardığı race (`List.Add` duplicate), `ApplyConfigurationsFromAssembly`'nin aynı entity configuration'ını iki kez çalıştırmasına ve duplicate `HasData` seed hatasına yol açıyordu.
- **Cross-test isolation smoke test** — `CrossTestIsolationSmokeTestsA` + `CrossTestIsolationSmokeTestsB` ikisi de aynı sabit GUID'yi aynı `OutboxMessages` tablosuna yazar; her biri `DB_NAME()` ile kendi DB'sinin `T_CrossTestIsolationSmokeTestsA/B_*` pattern'ine uyduğunu doğrular. Unique-DB izolasyonunun PK collision olmadan çalıştığını kanıtlar.
- **`IntegrationTestBaseSmokeTests.ConnectionString_IsNotEmpty_AfterInitialization`** — `SqlConnectionStringBuilder` çıktısı `Data Source=` formatında olduğundan, sabit `Server=` substring check'i format-agnostic kontrolle değiştirildi ve `Initial Catalog=T_` substring assertion'ı eklendi (unique DB pattern kanıtı).
- **BYPASS_LOG closing note** — PR #34 hot-fix'in kapanışı (başlık üstü nota eklendi); hot-fix satırı olmadığı için entry eklenmedi.

## Known Limitations / Follow-up
- **Lokal paralel execution OOM riski:** Env var yoksa (lokal dev) her test assembly kendi `MsSqlContainer`'ını açar. Paralel MSBuild (`-m:N` default=processor count) birden fazla container'ı aynı anda başlatabilir ve dev makinede bellek yetersizse OOM yaşanabilir. Mitigation: lokal geliştiride `dotnet test Skinora.sln --filter ~Integration -m:1` kullan veya env var ile shared server işaret et.
- **Service container log erişimi:** GitHub Actions `services:` container log'ları default olarak workflow step log'larına düşmez; SQL Server crash olursa tanı için `docker logs` step eklenir (şu an gerek yok).
- **İki smoke test sınıfı aynı amaçlı:** Cross-test isolation test'lerinin iki sınıfa bölünmesi xUnit'in farklı test collection'a atamasını garantilemek için. Future cleanup: tek class + `[Theory]` dönüşümü, collection attribute ile iki collection'a böl. Şimdilik plan kabul kriteri 8'i minimum işle karşılar.
